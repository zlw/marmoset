# P1 Fix Plan

## Scope

High-impact semantic divergence around records and traits:

- Record shape identity is order-sensitive in method/derive lookup.
- Structural record intent is not preserved when field order differs.
- Deterministic derive behavior depends on non-canonical field ordering.

## Concrete Work

1. Introduce canonical record shape normalization.
- Add a single normalization helper in `/Users/zlw/src/marmoset/marmoset-ml/lib/frontend/typecheck/types.ml`.
- Canonical rule for closed/open records:
  - Sort fields by name (stable lexical order).
  - Keep one field per name (last write wins already enforced earlier for literals/spread).
  - Keep row tail unchanged.
- Apply this normalization in:
  - record literal inference result,
  - annotation conversion result for `TRecord`,
  - substitution/unification boundaries where record values are produced.

2. Normalize record shape before trait impl registration/lookup.
- In `/Users/zlw/src/marmoset/marmoset-ml/lib/frontend/typecheck/trait_registry.ml`, key impls by normalized `impl_for_type`.
- In method lookup, compare normalized receiver type.

3. Normalize mangling for records.
- In `/Users/zlw/src/marmoset/marmoset-ml/lib/backend/go/emitter.ml`, make record mangling consume canonicalized fields so equivalent shapes map to one mangled name.

4. Deterministic derive behavior.
- Ensure derive-generated eq/show/ord/hash for records uses canonical field order.
- Keep hash combiner order tied to canonical order for deterministic output across equivalent shapes.

## Regression Tests

1. Reordered record literal should resolve derived trait methods.
- Example:
  - `type point = { x: int, y: int }`
  - `derive show for point`
  - `let p = { y: 2, x: 1 }`
  - `puts(p.show())`
- Must succeed and print canonical field order format.

2. Reordered aliases remain compatible.
- `type point = { x: int, y: int }`
- `type vec = { y: int, x: int }`
- Assignments and trait calls must behave identically.

3. Deterministic hash/eq/ord.
- Same logical shape/value emitted from different field orderings must compare/hash consistently.

## Exit Criteria

- Structural record equivalence no longer depends on source field order.
- Trait method dispatch for records respects structural semantics.
- Derive output for records is deterministic across equivalent shapes.
