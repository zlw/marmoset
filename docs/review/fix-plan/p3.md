# P3 Fix Plan

## Scope

Lower-severity but important maintainability and future-proofing work:

- Lexer/parser robustness around edge tokens and record/hash parsing mode.
- AST location model not rich enough for strong LSP tooling.
- Trait-object / existential representation is not yet first-class in compiler IR.
- Modules/FFI policy needs explicit constraints before implementation expands.

## Concrete Work

1. Lexer robustness for invalid dot/spread sequences.
- In `/Users/zlw/src/marmoset/marmoset-ml/lib/frontend/syntax/lexer.ml`, ensure illegal token paths always advance the cursor.
- Keep token position accounting correct for multi-char operators.

2. Record/hash parsing cleanup.
- Refactor record/hash literal parser to a single entry loop with explicit mode lock:
  - mode chosen after first entry,
  - later mixed-mode entries produce direct syntax error.
- Preserve current semantics (punning/spread/field access behavior).

3. Spans and file identity on AST nodes.
- Extend AST node metadata beyond start offset:
  - `file_id`,
  - `start_pos`,
  - `end_pos`.
- Thread through parser and typechecker so diagnostics and future LSP can reference exact ranges.

4. First-class trait-object/existential design note before implementation.
- Add a focused design section in canonical docs for future dynamic trait values:
  - representation shape,
  - erasure rules,
  - operation limits (`eq/hash/show` semantics on existential values),
  - module/FFI boundary constraints.
- No runtime implementation in this phase; this is to prevent incompatible ad-hoc designs later.

5. Modules/FFI guardrail decisions (documentation + checks).
- Lock “one Go package per build” as initial modules backend policy.
- Document ABI mapping constraints for potential extern/FFI types before exposing feature.

## Regression Tests

1. Lexer must terminate cleanly and produce stable errors on malformed `.`/`..`/`...` inputs.
2. Mixed record/hash literal forms must error deterministically.
3. Diagnostics should include range (start-end) once span metadata lands.

## Exit Criteria

- Parser/lexer edge cases are deterministic and maintainable.
- Compiler has range-capable source mapping.
- Future trait-object/modules/FFI work has explicit representation constraints, reducing rewrite risk.
